Git & GitHub Glossary

Repository (Repo)

A storage space where your project lives, containing all files and their version history.

Commit

A snapshot of your repository at a specific point in time with a descriptive message.

Branch

A parallel version of a repository that allows you to work freely without affecting the main project.

Merge

The process of combining changes from one branch into another branch.

Pull Request

A method of submitting contributions to a project, requesting that changes be reviewed and merged.

Clone

Creating a local copy of a remote repository on your computer.

Fork

Creating a personal copy of someone else's repository on your GitHub account.

Remote

A version of your repository hosted on the internet or network, such as GitHub.

Staging Area (Index)

A place where changes are gathered before committing them to the repository.

Push

Sending your committed changes to a remote repository like GitHub.

Pull

Fetching and integrating changes from a remote repository to your local repository.

Fetch

Downloading new data from a remote repository without integrating it into your local work.

Conflict

A situation where changes in different branches cannot be merged automatically and require manual resolution.

README

A markdown file that explains your project, how to use it, and any other relevant information.

Issue

A way to track tasks, enhancements, bugs, or questions related to a project on GitHub.

Action

Automated workflows that run on GitHub to build, test, or deploy your code.

Release

A packaged version of your project, often associated with a tag, for distribution.

License

A file that specifies the terms under which your project can be used, modified, and shared.
